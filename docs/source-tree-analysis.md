# Source Tree Analysis

- `config/` — strategy/runtime configuration
- `core/` — shared utilities and primitives
- `data/` — datasets and generated artifacts
- `doc/` — legacy docs
- `docs/` — notebooks and generated workflow outputs
- `scripts/` — helper scripts for automation
- `tests/` — automated tests
- `trading/` — trading logic, rules, and execution
- `bootstrap.py` — bootstrap entry
- `scheduler.py`, `trade.sh`, `download.py`, `validate.py` — orchestration and maintenance commands
